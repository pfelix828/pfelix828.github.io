InterviewReady

Live App

What This Project Does

Paste a job description URL and InterviewReady generates a tailored practice session — technical coding questions, case studies, and behavioral questions calibrated to the specific role, company, and seniority level. Answer in-app and get immediate AI-evaluated feedback with scoring and coaching.

It’s the tool I wanted when I was prepping for interviews and couldn’t find: something that adapts to the actual job instead of recycling the same generic question bank.

How It Works

Setup

Paste a job posting URL (or the raw text). The app auto-extracts the company name, role title, seniority level, and technical skills from the JD. Pick an interview type — Technical (coding), Case Study, or Behavioral — and start.

Question Generation

Questions are generated by Claude Sonnet, drawing from a 600+ question bank in Supabase and adapting to the target company and role. Technical questions always include sample data (SQL tables with column types, Python DataFrames with sample rows). Every question is scoped to be answerable in 2-3 minutes with only the provided data.

Quality rules are strict: one task per question, difficulty matched to JD seniority, company-specific tailoring (references to real products, never competitors), and a hardcoded skills allowlist so only genuinely codeable skills appear in technical mode.

Answer Evaluation

Answers are evaluated by Claude Sonnet (temperature 0) with structured feedback. Technical answers get correctness scoring. Behavioral answers are evaluated against STAR framework criteria. Feedback streams in real-time with a score, coaching notes, and specific suggestions.

Progress Tracking

The app tracks skill readiness across sessions using a multiplicative formula (score quality x practice volume). A skills readiness dashboard shows per-skill bars, score trends over time, best answers, and weak areas with targeted practice links. Gap analysis compares your resume against a JD to identify skill matches, gaps, and talking points.

Architecture

Frontend

Next.js 16 with App Router, React 19, Tailwind + shadcn/ui, and Zustand 5 for state management. Dark/light mode. Mobile-responsive with icon-only action bars and 44px tap targets on mobile.

Backend

All AI calls are server-side (no API keys exposed to the client). Three model tiers: Claude Sonnet for question generation and answer evaluation, Claude Haiku for hints, clarification, and JD parsing. Groq for voice transcription.

Auth & Data

Supabase Auth (Google OAuth + email magic link) with row-level security on all tables. Guest mode with cookie-based bypass for trying the app without an account. All 10 API routes validate authentication and input (Zod schemas). Rate limiting at 50 requests/hour per IP.

Security

Security headers (CSP, HSTS, X-Frame-Options, nosniff), SSRF protection on URL fetching (blocks private IPs/localhost), auth callback redirect validation, and userId ownership checks on all data access.

What I’d Do Differently

  • Question classifier — Collecting thumbs up/down feedback now to train an XGBoost model that auto-filters low-quality generated questions
  • User-based rate limiting — Currently IP-based, which is rough for shared networks
  • Longer function timeouts — Vercel Hobby plan caps at 10 seconds, which occasionally truncates complex evaluations

Stack

Next.js | React | TypeScript | Tailwind | shadcn/ui | Zustand | Claude API | Supabase | Vercel
